# local-media-vault 작업 지침

## 프로젝트 목적

- 같은 Wi-Fi 안에서만 접근하는 개인용 로컬 미디어 업로드 서버를 만든다.
- 모바일 웹에서 iPhone 원본 사진과 영상을 업로드하고, J3 같은 로컬/외장 저장소에 저장한다.
- 클라우드 저장소, 외부 공개 서버, 제3자 동기화 서비스에 의존하지 않는다.

## 개발 규칙

- 커밋 메시지는 항상 한국어로 작성한다.
- 기존 구조와 설정을 우선하고, 요청 범위를 벗어난 리팩터링은 피한다.
- 개인 미디어 파일, 토큰, 비밀번호, 실제 저장소 경로, 운영 환경 값은 커밋하지 않는다.
- 로컬 저장소 경로는 환경 변수나 설정 파일 예시로만 다룬다.
- 대용량 업로드는 원본 보존을 우선하고, 썸네일 생성 실패가 업로드 실패로 이어지지 않게 설계한다.

## 검증 기준

- Docker로 서버를 띄운 뒤 같은 Wi-Fi 모바일 브라우저에서 접속 가능해야 한다.
- 사용자별 권한이 API와 UI 양쪽에서 동일하게 적용되어야 한다.
- 저장 경로 템플릿은 허용 루트 밖으로 벗어날 수 없어야 한다.
- iPhone 원본 사진, HEIC, JPG/PNG, MOV/MP4 업로드를 검증한다.
